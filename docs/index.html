Romi Autonomous Robot — ME 405

Project Overview

This project programs a Pololu Romi robot to autonomously complete a play-track course at Cal Poly SLO for ME 405. The robot follows a black line using an 8-sensor IR array, estimates its position and heading using a Luenberger state observer fused with IMU data, and detects obstacles via a front-mounted bump sensor.

Hardware

  • STM32 L476RG microcontroller (Shoe of Brian)
  • Pololu Romi chassis with dual DRV8838 motor drivers
  • BNO055 IMU (I2C, IMU fusion mode)
  • QTR-8A analog IR line sensor array
  • Quadrature wheel encoders
  • HC-05 Bluetooth module (UART4)
  • Front bump switch (obstacle detection)

Software Architecture

All control runs inside a cooperative round-robin scheduler (cotask). Six tasks share state through Share and Queue objects:

  • task_motor (x2): PI velocity control, 31ms period
  • task_line: PID line following, 22ms period
  • task_state: Luenberger state observer, 31ms period
  • checkpoint_task: High-level navigation FSM, 31ms period
  • task_user: Bluetooth command interface

State Estimation

The observer tracks four state variables:

  • s: forward displacement (mm)
  • psi: heading (rad)
  • omL: left wheel angular speed (rad/s)
  • omR: right wheel angular speed (rad/s)

Update law: x_{k+1} = AD * x_k + BD * u*_k

Inputs fused each tick: encoder positions, BNO055 heading and yaw rate, and motor effort values.

Line Following

The QTR-8A driver computes a centroid position in mm from the sensor center. A PID controller (Kp=6.0, Ki=0.7, Kd=0.9) converts this error into a steering correction, filtered through an EMA (alpha=0.45) before reaching the motor tasks.

Results

The robot successfully navigated the course. Key lessons learned:

  • EMA filtering on steering output dramatically reduced oscillation
  • IMU fusion in IMU mode gave stable heading despite motor noise
  • Lost-line hold logic (4 tick hold + 85% decay) handled sharp curves
